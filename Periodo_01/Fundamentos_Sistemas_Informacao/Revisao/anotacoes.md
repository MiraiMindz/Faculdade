## Aula 01

O que é um sistema?

> Conjunto de partes que fazem com que um todo (o sistema) atinja seu objetivo.

> Um conjunto de elementos inter-relacionados onde cada um desempenha uma função para contribuir com que o objetivo do sistema seja atingido de forma coordenada e integrada.

A coordenação das partes faz com que o sistema se sobre saia e execute sua função devidamente.

Cada componente de um sistema tem:
- Entrada: recebe uma algo de um outro componente
- Processamento: processa e transforma a entrada
- Saida: gera novas coisas para serem passadas para outro componente.
- Controle: a parte responsavel por gerenciar o sistema.
- Retroalimentação (Feedback): ocorre quando parte da saida é reutilizada para ajustar a entrada ou o processamento.

```
Entrada ----> Processamento ----> Saida
^                   ^                 ^
|                   |                 |
\------------------/\-----------------/
            Retroalimentação
                Controle
```

> Toda parte *recebe* algo, *faz* algo (transforma/processa) e *transmite* como resultado alguma outra coisa.

O que é um sistema de informação?
- É um tipo de sistema cujo a finalidade é prover informação as empresas, através do processamento de *dados*.

Dados x Informação
- Dados:
    - São os elementos de entrada de um Sistema de Informação. 
    - Dado é um fato isolado, sem valor agregado. Forma bruta.
- Informação:
    - Tem utilidade pra quem recebe.
    - São dados organizados, eles tem valores agregados.
    - Resultado do processamento de um ou mais dados.

> o número de faltas de um aluno é um *dado*, após ser processado, e mostrar o total de faltas no semestre e se este aluno foi aprovado com frequência, será uma *informação*.

O processamento pode ser um cálculo, uma comparação, algo que transforma (cataloga/analisa).

O processamento deve ser lógico e significativo (devem-se existir regras, para tal), além de ser veloz, a velocidade é crucial, a informação deve ser provida o mais rapido possivel (momento oportuno)

o SI contém/dependende de pessoas.

O SI automatizado é aquele que contém tecnologia.

O sistemas manuais, são limitados (ineficientes).

O sistema automatizado pode prover:
- agilidade: a velocidade da informação é crucial
- produtividade: realização de tarefas (realizar taferas mais rápido)
- competitividade: provém informações estrategicas para vantagem no mercado
- otimização de custos: otimiza os custos, diminui os gastos e pode aumentar os lucros

Os níveis fundamentais de uma empresa:

1. Nivel Estratégico: Sistema de informação estratégico
    - toma *decisões* de longo prazo (anos), que precisam de informações.
    - Util para os Diretores e Presidentes.
    - determinam o rumo da empresa.
    - precisa de informações externas a empresa (concorrencia, mercado, etc...)
    - usar informação como diferencial competitivo.
    - as informações são Macrorrelacionadas (MAIE).
2. Nivel Gerencial: Sistema de informação gerencial
    - Nivel que coordena a operação.
    - toma *decisões* de médio prazo (meses), que precisam de informações.
    - Util para os Supervisores, Gerentes, etc...
    - As informações são agrupadas, a nivel de grupo
3. Nivel Operacional: SIstema de Processamento de transações
    - toma *decisões* de curto prazo (diárias), que precisam de informações.
    - operação básica da empresa, operações diárias.
    - a transação é a operação básica de uma empresa.
    - As informações acontecem no nivel/uso detalhado.
    - Util para o corpo tecnico

Um SI deve ter:
- Reponsabilidade
- Gerenciamento
- Desenvolvimento da capacidade
- Disciplina
- Processos de aperfeiçoamento contínuo
- Tecnologia (TI)

> a TI provém uma base de dados unica e informação para todos os niveis e setores.

Os componentes da TI:
- Hardware: Componente físico
- Software: Componente lógico
- Banco de dados: Armazena os dados e as informações
    - SGBD (Sistema Geral de Banco de Dados): Organiza, gerencia e processa os dados.
- Redes e tele-comunicações: Conectam os sistemas de computação, audio, imagem, video, etc...

Os Sistemas e as empresas sofrem influencia do ambiente, e devem se adequar a isso.

Os sistemas são hierarquicos e constituidos de outros sistemas (sub-sistemas) e de forma recursiva (níveis).

```
         Sistema (subsistema também)
            /-------------------\
            | Subsistema 01     |
Entrada --> |    Subsistema 02  | --> Saida
^           |  Subsistema 03    |         ^
|           \-------------------/         |
\-----------------------------------------/
                  Feedback
```

Teoria Geral de Sistemas (TGS)

- Classificação dos Sistemas:
    - Aberto: sofre influencia (positiva ou negativa), e influencia o meio em que está inserido. É o oposto do sistema Fechado.
    - Complexo: suas partes (sub-sistemas) são complexos e com muitas interconexões. É o oposto do sistema Simples.
    - Dinamico: responde rapidamente a uma alteração no ambiente que está inserido. É o oposto do sistema Estável.
    - Adaptavel: capaz de sofrer mudanças internas (componentes, entradas e saidas), em resposta a mudança no ambiente. É o oposto do sistema Não adaptavel.
    - Permanente: quanto maior o seu tempo de vida, maior será o retorno dado a quem invenstiu (+/- 10 anos). É o oposto do sistema Temporário.
    - Fechado: Tem pouca ou nenhuma interação com o ambiente. É o oposto do sistema Aberto.
    - Simples: possui poucos elementos (sub-sistemas) e suas relações são simplificadas e diretas. É o oposto do sistema Complexo.
    - Estável: mudanças no ambiente tem pouco ou nenhum impacto no sistema. É o oposto do sistema Dinamico.
    - Não Adaptavel: não sofre alterações internas com mudanças do ambiente. É o oposto do sistema Adaptavel.
    - Temporário: Não existirá por um longo periodo de tempo, e portanto o retorno é curto (1 mês ~ 1 ano). É o oposto do sistema Permanente.

- Funcionalismo: cada elemento desempenha uma função especifica para atingir o objetivo total do sistema.

- Holismo: Todos os sistemas são compostos de sub-sistemas, e seus elementos são inter-relacionados. Vale ressaltar que o todo não é a soma de todas as partes, você só explica um sistema entendendo todas as suas partes e suas inter-relações.

- Feedback (retro-alimentação): a saida de um sistema é reutilizada para ajustar a entrada ou o processamento.

- Homeostase (auto-regulação): é a tendencia de equilibrio de todo sistema, com o objetivo de manter a constancia do sistema durante dado periodo de tempo.

- Diferenciação e integração: o sistema deve ser capaz de se adaptar e reordenar (mudar) em resposta as mudanças externas do ambiente.

- Fronteiras: é o que separa o sistema e seus componentes do meio ambiente externo.


## Aula 02

SI = TI + RH (pessoas) + Procedimentos

TI = Hardware + Software + Redes + Dados

Hardware:
- Entrada & Processamento
- Armazenamento & Saida
- Processamento (CPU por exemplo)
- Comunicação (internet, bluetooth, etc...)

Software:
- Aplicativo:
    - Finalidades Gerais: Ex. Pacote Office
    - Finalidades Especificas: Ex. ERP, Contas a pagar
- Sistema (Básico):
    - Gerenciamento de Sistemas: Ex. Sistema operacional, sistemas embutidos (embarcados), banco de dados, drivers, etc...
    - Desenvolvimento de Sistemas: Linguagens de programação, compiladores, (tudo relacionado a desenvolvimento).

Armazenamento de Dados:

Os dados precisam ser armazenados para processamentos futuros, podemos armazenar arquivos, ou em tabelas relacionadas e estruturadas que contém esses dados.

Por exemplo: existem alunos que cursam determinadas disciplinas, podemos guardar as disciplinas e os alunos e uma tabela contendo uma relação entre cada aluno e suas disciplinas ou vice-versa.

A informação é contextual, a saida de um nível serve de dado para o próximo.

Banco de dados e o SGBD (Sistema Gerenciador de Banco de Dados)

exemplos de SGBD são: MySQL, PostgreSQL, SQLServer, SQLite, Oracle, etc...

Os bancos de dados garantem que os dados estejam integros, não redundantes e consistentes.

Eles ser aptos as todas as aplicações da empresa.

Redes = Sistemas de computação + Sistemas de comunicação

Os sistemas de comunicação contém:
- Meios de transimssão (meio fisico)
- Protocolos de comunicação (software), como: FTP, TCP-IP, HTTP/S, UDP, etc...

Classificação das redes:
- LAN (Local Area Network/Rede de Área Local): Os nós (computadores) estão geograficamente proximos, no mesmo predio ou casa por exemplo.
- MAN (Metropolitan Area Network/Rede de Área Metropolitana): Os nós estão no perimetro de uma cidade.
- WAN (Wide Area Network/Rede de Longa Distancia): Os nós estão geograficamente distantes (Cidades ou outros paises), as redes são ligadas por satélites, ondas de rádio, etc...; a Internet é um exemplo de WAN.

RH no SI:
- São pessoas que:
    - Gerenciam: Gerentes de TI, Gerentes de Projeto, Gerentes de CPD, etc...
    - Desenvolvem e mantém: analistas, programadores, projetistas, designers, etc...
    - Suportam: Tecnicos de Rede, banco de dados, etc...
    - Usuários: Pessoas que usam o sistema.

Procedimentos:
    - Toda empresa tem processos, e todos os processos tem procedimentos.
    - Eles devem ser explicitos, o funcionario deve ter fácil e rapido acesso a essa informação.

```
Dados de entrada --> Processamento --> Informação de saida
                           ^
                           |
                     Conhecimento
```

O conhecimento pode ser tácito (armazenado no cérebro da pessoa) ou explícito (armazenados em livros, procedimentos ou sistemas de computador).

Quando o conhecimento está contido no sistema, sua automação torna-se elevada.

Para todo e quaisquer sistema de informação devem se haver algumas qualidades, a informação deve ser:
- Precisa: não conter erros
- Confiavel: é dependente da fonte da informação
- Relevante: é importante para o tomador de decisão
- Em tempo: enviada no momento necessário
- Economica: Os custos não devem ser maiores que os beneficios (Custo X Beneficio)

Os dados e as informações dependem do contexto aplicado: "Os conceitos são relativos (Dados num contexto pode ser informação em outro)."

a informação de saida do SPT é o dado de entrada do SIG

a informação de saida do SIG é o dado de entrada do SIE

Tipos de SI:
- Funções Empresariais:
    - Produto ou serviço
    - Produção do bem ou serviço
    - Vendas ou Marketing
    - Recursos Humanos
    - Finanças e Contabilidade
Uma empresa precisa de SIs em todas as suas areas, e eles podem ser *separados* ou *integrados*

Os sistemas para atender a cada função devem (ou deveriam) prover informações em todos os níveis.

Sistema Financeiro:
- SPT: Sistemas que monitorem o fluxo de recursos realizados pelas transações como cheques e pagamentos a fornecedores.
- SIG: Sistemas que ajudem o gerente a supervisionar e controlar recursos financeiros.
- SIE: Sistemas que estabeleçam metas de investimento prevendo desempenho financeiro.

Relacionamento entre os tipos de sistemas:

```
SPT --> SIG --> SIE
        |        ^
        |        |
        \-->SAE--/
```

Sistema de Informação de Apoio a Decisão (SAE): Trabalha com problemas não estruturados.

Controle e desempenho de sistemas

Os sistemas devem ser sempre monitorados, de forma a avaliar: Se o nível de desempenho se mantém estável (ao longo do tempo e suas inúmeras execuções).

"O desempenho é medido pela eficiencia, um sistema eficaz atinge os objetivos; e um sistema eficiente só é eficaz quando atinge os objetivos com o menor tempo possivel".

Fatores que afetam o nível de eficiencia de um sistema:
- Meio ambiente: Ex. Refrigeração do hardware
- Problemas no hardware: Ex. Disco cheio
- Nivel de acesso e uso do sistema: Ex. mais lento o processamento
- Redes de comunicação de dados: Ex. congestionamento na rede
- Rede elétrica com picos e ausencia de corrente: Queda do sistema ou Banco de Dados.

A parada de um Sistema de Informação pode causar grande dano a uma empresa. Se o tempo de inoperância for grande, o risco de prejuízo cresce.

A solução é monitorar o uso dos sistemas de informação, ser proativo, ou seja, identificar possíveis elementos que podem influenciar sua performance.

## Aula 03

SI e suas fases:
1. Processamento de dados
    - Entre 1945 ~ 1970
    - SPT: Automação das rotinas
2. Relatórios administrativos 
    - Entre 1960 ~ 1980
    - SIG: Relatórios administrativos com informações pré-definidas para tomada de decisão.
3. Apoiando as decisões
    - Entre 1970 ~ 1990
    - SAD: Relatórios para apoio a decisões não estruturadas (não programadas).
4. Apoiando o nível estratégico
    - Entre 1980 ~ 1990
    - SIE: Sistema especialista. Apoio à produtividade do usuário final.
5. Conectando a empresa
    - Entre 1990 ~ 2000
    - A empresa globalizada na internet.

- Sistema de Processamento de Transações (SPT)
    - Caracteristicas:
        - Enorme capacidade de processamento.
        - Primeiras aplicações: controle bancário, contabilidade, financeiro (pagar e receber).
        - São críticos para as empresas, se deixam de funcionar.
    - Visam:
        1. Oferecer suporte aos processos e operações de rotina.
            - (Registram as transações rotineiras que sustentam o funcionamento da empresa.)
        2. Informações para gerir as operações.
        3. Assegurar a eficácia e a eficiência dos processos.
    - As transações: São o registro de um evento que o sistema precisa responder. Os sistemas administrativos básicos atendem ao Nível Operacional, eles registram eventos.
    - Tipos de processamento
        - Batch (Processamento em Lote)
            1. As transações ficam acumuladas por um tempo.
            2. O processamento opera com um lote.
            - Caracteristica: atraso entre a ocorrência do evento (e seu registro) e o processamento.
            - Exemplo: Folha de Pagamento, ela sempre ocorre num determinado dia do mês.
        - Online:
            1. As transações são processadas imediatamente no ato da sua ocorrência.
            2. Atualiza todos os registros afetados pela transação.
            - Exemplo: Sistema de vendas online; o cartão de debito, o valor é descontado de sua conta no mesmo momento em que você realiza a compra.
- Sistema de Informações Gerencias (SIG)
    - Caracteristicas:
        - Necessidades de organizar melhor a INFO.
        - Focar na colaboração ao processo decisório:
            - Operacionais -> supervisão.
            - Gerenciais -> Gerentes, Diretores.
        - Produzem relatórios periódicos.
    - Visam
        1. Subsidiar os gestores com informações úteis e em tempo
            - Informações agrupadas -> ajudam nas decisões diárias.
        2. Atender as atividades de monitoramento, controle, tomada de decisão e procedimentos administrativos dos gestores médios.
            - EXEMPLO: Investimos a sobra de R$ em estoque ou Marketing para expandir os negócios.
- Sistema de Apoio à Decisão (SAD)
    - Caracteristicas:
        1. Apoiam os gerentes de nível médio a resolverem problemas não estruturados.
            - Exemplo: Seleção de preço e rota em CIA aérea.
        2. Focam as decisões NÃO USUAIS, que se alteram com rapidez e não são facilmente especificadas com antecedência.
        3. Pode usar informações externas, de mercado, de governo, índices.
        4. Geram informações analíticas e modelos para interpretação de dados.
        5. Oferecem suporte interativo, permitindo que usuário faça simulações.
            1. Incluir novas perguntas, novos dados e novos cenários.
            2. Sofisticadas técnicas de análise de dados.
            3. É diferente das respostas pré-programadas do SIG. 
- Sistema de Informações Estratégicas (SIE)
    - Caracteristicas:
        - Necessidades de subsidiar ao alto escalão a definir os rumos da empresa no mercado – estratégia.
        - Apoio a questões estratégicas e tendências de longo prazo (empresa e ambiente externo).
        - Recursos gráficos. Múltipla visão dos dados.
    - Visam:
        1. Subsidiar a alta direção a obter vantagens competitivas.
        2. Compatibilizar as mudanças do ambiente externo com a capacidade da organização.
        - Exemplo: Abrimos uma nova fábrica nessa cidade?

Novo nível, Gestão do conhecimento

1. SIG
2. SIE
3. STC (Sistema de trabalhadores do conhecimento)
4. SPT

Sistema de traabalhadores do conhecimento:
- Auxiliam a empresa na busca, integração e implantação de novas tecnologias que *agreguem valor ao negócio*.
- Ajudam no controle do fluxo de documentos.
- As ferramentas desse nível ajudam:
    1. No processo de criação da informação (STC).
    2. Sistemas colaborativos: aumentam a comunicação e a produtividade da equipe.

> **Sistemas colaborativos** são softwares e aplicativos que operam em redes, preferencialmente na nuvem, e têm o objetivo de otimizar o trabalho em equipe, a troca de informações e o fluxo de ideias e materiais, como arquivos, planilhas, apresentações, documentos de texto e outros.

Outros Tipos de Sistemas
- Visam simular a forma como especialistas trabalham, por exemplo: Um médico ao fazer um diagnóstico. (A ideia é simular os processos mentais que ele usa para realizar tal tarefa.)

- Sistema de Gerenciamento de Documentos (SGD)
    - Sistemas que permitem a digitalização de documentos, não só para reduzir a necessidade de espaço físico de armazenamento, mas também para facilitar as pesquisas nos documentos.

- Sistema de Gestão Integrada (ERP)
    - Benefícios:
        1. Integração das operações internas da empresa.
        2. Ganhos de eficiência.
        3. Aumentos de controle sobre processos.
        4. Informações de qualidade sobre a operação dos negócios, no momento certo da decisão.
    - Observações:
        1. Implementação não é fácil, ela exige a reorganização de processos e uma mudança cultural.
    - Visam integrar a empresa em um único sistema, comercializado de forma modular e customizável.
    - Base de dados integrada.
    - Integram informações operacionais (SPT), gerenciais (SIG e SAD) e estratégicas (SIE).

## Aula 04

E-Business (Negócios Eletronicos):
- São negócios eletronicos feitos pela internet, ocorrem quando um empresa usa a tecnologia digital e a internet para executar os principais processos de se negócio.
- Possibilita a integração entre:
    1. a empresa
    2. os clientes
    3. os fornecedores
- Além de possibiltar a mobilidade, a rapidez e por sua vez a competitividade.
- Caracteristicas:
    - É o modelo de negócio em expansão.
    - Necessita investimento ou tecnologias compartilhadas
    - Acomoda bem os conceitos de SAD e SIE (Dados Externos)
        - Usa dados externos para tomar decisões, usando a tecnologia de internet.
        - Ex: Gestão em viagens acompanham os negócios.

A partir dos anos 2000, a empresa com o auxilio da internet conseguiu ir para além de suas fronteiras fisicas.

Os tipos de relações no e-Buisiness:
- B2B (Business to Business): Ocorre quando há relação entre duas empresas de um mesmo ramo ou de areas distintas.
    - > Em um cenário extremamente competitivo, enxergar outras empresas que atuam na mesma área como concorrentes apenas, se tornou uma visão ultrapassada. Hoje a parceria é fundamental para o crescimento de ambos os lados. A competição ainda existe, mas a cooperação é um passo importante. Várias empresas abrem espaços para parceiros em suas intranets a fim de compartilhar conhecimento.
- C2C (Consumer to Consumer): Ocorre quando consumidores lidam diretamente com outros consumidores através de um serviço.
    - > O tipo de transação C2C se popularizou bastante nos últimos anos. Nessa classe os consumidores lidam diretamente com outros consumidores, como no site [Mercado Livre](http://www.mercadolivre.com.br). Com a popularização de redes sociais, os aplicativos de compra e venda de produtos nessas redes começa a surgir e promete ser uma área promissora. O Facebook, por exemplo, já conta com um aplicativo que pode ser integrado a páginas pessoais ou até mesmo de empresas, facilitando o processo de compras.
- B2C (Business to Consumer): Ocorre quando o negócio provém utilidades para o consumidor, vendas por exemplo;
    - > Representa o comércio e transações comerciais entre uma empresa e um cliente. É o exemplo mais fácil de ser entendido, uma vez que já se tornou comum realizar compras no ambiente virtual.
      >
      > Um dos exemplos mais bem sucedidos hoje de B2C é a empresa americana "[Amazon](http://www.amazon.com)", que inicialmente trabalhava com o comércio de livros através de compras realizadas pela internet. Contudo, ao enxergar novas oportunidades para seu negócio, o escopo de negócio da empresa foi redefinido e ela passou a atuar em áreas de mercado completamente diferentes de sua área inicial.
      >
      >> Isso só foi possível graças a avanços em tecnologias de suprimentos, adicionando a capacidade de análise de estoque de fornecedores, e no avanço de tecnologias de análise de perfil do cliente.
      >>
      >>> *"Isso só foi possível devido a poderosa integração de sistemas utilizados, como os de atuação em processos de e-SCM (Supply Chain Management/Gestão da Cadeia de Suprimentos) a fim de controlar os produtos a venda, permitindo que a empresa vendesse produtos que estariam estocados em seus fornecedores, aumentando assim o leque de produtos oferecidos e eliminando a necessidade de grande estocagem local. Sistemas como os do tipo e-CRM(Customer Relationship Management/Gestão de Relacionamento com o Cliente) permitem a análise do perfil do cliente, oferecendo a ele produtos de interesse. Aliados a um bom sistema de BI, foi possível tomar decisões estratégicas com menor risco, o que foi responsável pelo sucesso da empresa, que hoje entrega seus produtos mundialmente."*
- C2B (Consumer to Business): Ocorre quando o consumidor gera um produto para a empresa, por exemplo num concurso de marketing os consumidores geram videos e propagandas, das quais a empresa pode adiquirir.
    - > permite que os negócios extraiam valor dos clientes, e vice-versa – assim o negócio lucra com a vontade do consumidor em estabelecer o seu preço, contribuir para o negócio de alguma forma, ou fazer promoção à empresa enquanto goza de pagamentos diretos ou reduções de preços ou descontos, por exemplo.
      >
      > Uma manifestação do C2B é a criação de valor gratuita como por exemplo um consumidor coloca uma crítica positiva no site de um produto, ou quando fornece uma boa ideia para um novo produto.
      >
      > Num panorama mais negocial, o modelo C2B pode contemplar ainda a situação de um cliente individual oferecer, efetivamente, produtos/serviços a empresas que podem vir a comprá-los – por exemplo, quando uma empresa abre um concurso para a criação de um logótipo e vários designers concorrem com propostas que podem vir a ser compradas.
- B2G (Business to Governement): Ocorre quando um negocio tem relação com entidades governamentais, como na licitação de projetos, ou parcerias público-privadas por exemplo.
- B2E (Business to Employee): Ocorre quando a empresa se relaciona com seu funcionário (empregado e empregador), por exemplo, no fornecimento de cursos de formação.

O Crescimento dos SI nas empresas:

- Anos 50: Mudanças tecnicas
- Entre os anos 60 e 70: Controle Gerencial
- Entre os anos 80 e 90: Atividades Institucionais Centrais
- Entre os anos 2000 e 2005: Fornecedores e Clientes, expansão para além das fronteiras fisicas da empresa com o auxilio da internet.

O poder da internet e do E-Business nas empresas

No passado, por questões de custo ou inviabilidade técnica muita empresa grande, realizava a integração de suas filiais através do processamento *BATCH*.
- Cada filial tinha uma estrutura de processamento local
- De tempos em tempos enviavam seus dados a matriz, que os centralizava

Tinha alto custo com redundância, além da possibilidade de inconsitencia dos dados, devido a divergencia entre as bases locais e as bases centralizadas.

> A internet possibilitou a integração do sistema, que passa a ser online, com base única e menor custo.

O SAD e o SIE na era da internet

- O Sistema de Apoio a Decisão (SAD) e o Sistema de Informações Estratégicas (SIE), por usarem fontes externas e serem destinados a gestores, que viajam, naturalmente se adaptam ao contexto WEB.
- A consolidação das tecnologias da INTRANET/EXTRANET e do E-COMMERCE, criam a estrutura para migração da plataforma
    - INTRANET – Rede privada, usa os recursos da internet.
    - EXTRANET – Rede privada, usa os recursos de telecomunicações para permitir o acesso remoto a empresa.

| :--------------                 | :------: | :------: | :------: |
| Acesso restrito                 | Não      | Sim      | Sim      |

Algoritmos Genéticos:
- Usado em processos de resolução de problemas de otimização, que sejam difíceis de resolver por técnicas tradicionais (Sistema de busca para problemas complexos, com uso de muitas variáveis).
- É o que chamamos de escolha da solução ótima.
- Porque Genéticos?
    - Algoritmos heurísticos de busca inspirados na teoria da evolução, seleção natural e nos processo biológicos
    - Processo iterativo de seleção da população de candidatos.
- O que é otimização?
    - Mecanismo de análise de decisões complexas, envolvendo a seleção de valores para variáveis para quantificar performance e medir qualidade das decisões
- Aplicações
    - Algoritmos genéticos diferem dos algoritmos tradicionais de otimização.
        1. Se baseiam em uma codificação do conjunto de soluções possíveis, e não nos parâmetros de otimização em si.
        2. Os resultados são apresentados como uma população de soluções e não solução única
        3. Não necessitam de nenhum conhecimento derivado do problema, apenas de uma forma de avaliação do resultado
        4. Usam transições probabilísticas e não regras determinísticas.
- Os algoritmos genéticos são úteis para situações nas quais milhares de soluções são possíveis e precisam ser avaliadas para a escolha de uma solução ótima.
- Algoritmos Genéticos (AG), são utilizados em processos de resolução de problemas de otimização de difícil resolução por técnicas tradicionais (maximização).
-  Os AGs são apropriados para problemas de otimização complexos, que envolvem muitas variáveis e um espaço de soluções de dimensão elevada.
- O software de algoritmo genético utiliza um conjunto de regras de processo matemático que especificam como a combinação de componentes deve se feita.
- Exemplos:

- O objetivo dos SE's é de captar o conhecimento de um especialista em um determinado campo, representar este conhecimento numa base e transmiti-lo ao usuário, permitindo-lhe obter respostas a perguntas relacionadas à base de conhecimento do sistema.
- Os especialistas transferem o seu conhecimento ao Software.
- Base do Conhecimento
    - Armazena o conhecimento do domínio do problema
    -'Armazena sob a forma de REGRA
    - Se `<SITUAÇÃO>` ENTÃO `<CONCLUSÃO OU AÇÃO>`
    - Exemplo:
        - REGRA 01:
            - SE o carro não ligar,
            - ENTÃO, o problema pode residir no sistema elétrico.
        - REGRA 02:
            - SE o problema pode estar no sistema elétrico e a voltagem da bateria está apresentando um valor abaixo de 10 volts,
            - ENTÃO a falha reside na bateria que se encontra ruim

    - Memória de Trabalho
        - É o elemento que armazena o conhecimento concreto, ou seja, os fatos antes do processo de inferência e as conclusões seguintes.
        - Esta memória é de caráter transitório pois, novos fatos estão sendo acrescentados continuamente ou fatos existentes são apagados.
            - Exemplo: duram o processo de consulta, até que se chegue a uma conclusão
        - Armazenam as perguntas já respondidas – evita responder a mesma pergunta 2 vezes.
        - Armazena a linha de raciocínio usada.
    - Mecanismo ou Motor de Inferencia
        - O mecanismo de inferências (MI) é o processador ou interpretador de conhecimento, sendo considerado o coração do SISTEMA ESPECIALISTA.
        - Este é responsável por buscar, selecionar e avaliar as regras que foram coletadas na base de conhecimento.
        - A sua principal função é combinar o conhecimento abstrato contido na base de regras, com o conhecimento concreto armazenado na memória de trabalho, inferindo conclusões e gerando novos fatos.
- As aplicações típicas de sistemas especialistas são atividades que envolvem diagnósticos
- Desvantagens
    - Difícil absorção de todo o conhecimento necessário aos especialistas.
    - Falta de flexibilidade.
    - Impossibilidade de explicar detalhes aos usuários. (limitando ao conhecimento expresso no sistema).
    - Limitada capacidade de aprendizado.
